# Методические заметки: FA Quest Lab Bot

## Новая логика

Бот теперь работает не как один фиксированный квест, а как платформа для нескольких квестов.

Студенты начинают с кода:

```text
MISSING2026
MR2026FA
```

После ввода кода бот загружает нужный JSON-сценарий.

## Рекомендуемая организация кодов

Для одного и того же квеста можно сделать разные коды для разных групп:

```text
MR-GROUP1
MR-GROUP2
MR-GROUP3
```

Все они могут вести к одному файлу `quests/missing_reel.json`.

Это удобно, если вы хотите:
- разделить потоки студентов;
- не показывать всем один общий код;
- проводить один квест в разные дни;
- быстро отключать конкретную группу через `"active": false`.

## Что менять чаще всего

### 1. Коды

Файл:

```text
quest_catalog.json
```

### 2. Тексты квестов

Папка:

```text
quests/
```

### 3. Ответы

Внутри каждого файла квеста:

```json
"accepted_answers": ["answer", "variant 1", "variant 2"]
```

Для Telegram-бота лучше добавлять все ожидаемые варианты:

- разные регистры не нужны, бот сам их нормализует;
- добавляйте русские варианты, если допускаете кириллицу;
- добавляйте единственное и множественное число;
- добавляйте распространённые синонимы.

## Что не нужно делать

Не создавайте нового Telegram-бота для каждого квеста. Достаточно добавить новый JSON-файл и новый код в `quest_catalog.json`.

## Пилотное тестирование

Перед занятием проверьте:

1. `/start`
2. ввод кода `MISSING2026`
3. `START`
4. `I AGREE`
5. правильный ответ на L1
6. неправильный ответ на L2
7. `HINT`
8. `SKIP2`
9. `/reset`
10. `/code`
